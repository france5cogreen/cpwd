# **cpwd**

Simple tool for automatic local password change

A robust multi-threaded Python script designed for network administrators to automate batch password rotations across multiple network devices (Cisco, Fortinet, Palo Alto, SonicWall, Alcatel, Tiesse).

This tool focuses on safety: it generates cryptographically secure passwords, tests connections before applying changes, verifies the new credentials, and attempts an automatic rollback if the new password is not accepted.

## **Key Features**

* **Safe by Default (Dry-Run):** The script runs by default in "dry-run" mode unless you use the \--wet-run option. You can use it to test connection, authentication and commands without modifying any device configurations.  
* **Highly Concurrent:** Uses Python's ThreadPoolExecutor to process multiple IP addresses simultaneously, drastically reducing execution time for large networks.  
* **Automatic Rollback:** If the script successfully changes a password but fails to log in with the new credentials, it attempts to revert to the old password automatically to prevent device lockouts.  
* **CSPRNG Password Generation:** Uses Python's secrets module to generate mathematically unpredictable, highly complex passwords (uppercase, lowercase, digits, and special characters).  
* **Comprehensive Reporting:** Automatically categorizes results (Success, Warning, Unreachable, Error, Critical) and outputs both a human-readable text report and a structured JSON file.  
* **Personalized number of workers:** Configure parallel threads to fit your needs (default 4).

## **Usage**

First, create a text file (ex. ip\_list.txt) containing the IP addresses of your target devices, one per line. You can use \# to comment out lines.

### **Example 1\. Dry-Run (Recommended first step)**

Launch the script with:

python change\_pwd.py ip\_list.txt

This command will execute the script in dry-mode. This will NOT modify the credentials but test your connectivity, credentials and that the script has the right commands for your O.S. version. *(The script will prompt you for the device type, access username and current password).*

### **Example 2\. Wet-Run (Actually change passwords)**

Apply the changes across all devices:

python change\_pwd.py ip\_list.txt \--device-type alcatel \--workers 8 \--wet-run

### **Example 3\. Set the Same Password for All Devices**

Generate a single secure password and apply it to every IP in the list:

python change\_pwd.py ip\_list.txt \--wet-run \--shared-password

## **Argument Description**

* file\_ip : *(Required)* Path to the text file containing target IP addresses.  
* \--wet-run : Executes the actual configuration commands on the devices. (Default: False / runs Dry-Run)  
* \--workers N : Number of parallel threads to use (Default: 4\)  
* \--device-type : Specify the device type (cisco, fortios, paloalto, sonicwall, alcatel, tiesse). Prompted interactively if omitted.  
* \--log-file FILE : Save console execution logs to a specific file (Default: None)  
* \--output FILE : Specify the name for the output report files (.txt and .json). (Default: output\_passwords.txt)  
* \--shared-password : Generate only one password and apply it to all devices. (Default: False)

## **⚠️ Security Disclaimer**

This script handles sensitive data, use it only after accurate tests on your environment and architecture. The output files generated by this script will contain the newly generated plaintext passwords for your devices, but we highly recommend to use also the \--log-file option.

## Known Limitations & Roadmap

- **Palo Alto**: password change uses interactive SSH prompts, which can be
  fragile on some PAN-OS versions. A future version will use the XML API.
- **Alcatel**: implementation of the flash-synchro and certified/not certified managing commands
- **Checkpoint**: command syntax assumes Gaia R80+. Older versions may need `cpconfig`.
- **Interactive shell** (manual recovery mode): Linux only — `tty`/`termios` not
  available on Windows.
- [ ] SSH port override (e.g. Huawei sometimes uses 22222)
- [ ] `--ask-new-password` flag to supply the new password manually instead of generating it
- [ ] Regex-based output parsing to detect vendor error messages post-command
- [ ] `--verify-only` mode: test login without changing anything
- [ ] Config backup before password change (e.g. `show running-config` → file)
- [ ] Implementation of the -v option to show the actual script version
- [ ] Integration with Rundeck to provide new inventory sources, other than a file
